The Big Idea

One platform. Two interfaces. Zero JDE training required.

The JDE PO Agent bridges the gap between how your team communicates and how JD Edwards EnterpriseOne actually works — without bypassing a single business rule.

What it is
A Model Context Protocol (MCP) server — TypeScript, containerised — that exposes 18 purpose-built tools across five layers: dynamic discovery, data dictionary, PO CRUD, supporting lookups, and approval workflow. Comes packaged with a REST API and a React web dashboard.
What it does
Translates natural language requests — "create a PO for Acme Corp, 100 units of PUMP-2000 at branch M30" — into validated JDE operations: supplier lookup, item availability check, orchestration call, PO creation, and approval routing. All through the JDE AIS REST API.
What you get
Procurement teams create and track POs through conversation. Approvers see a live queue — in chat or the web dashboard — and approve with a single reply. JDE business logic (P43, F4301, F4311) runs exactly as configured. No workarounds, no shadow processes.

How It Works

Four steps from intent to approved PO.

The agent orchestrates every step invisibly — validation, creation, and approval — while your JDE configuration stays fully in control.

1
State intent in natural language
A team member types a request — "create a PO for Acme Corp, 100 units of PUMP-2000 at branch M30" — to an AI agent or through the web dashboard's chat interface. No form fields, no JDE screen knowledge required.
2
Agent validates supplier and item availability
The agent calls the supplier lookup tool to resolve the supplier record against JDE's Address Book, then checks item availability at the requested branch via the AIS Data Service. Any issues are surfaced in plain language before anything is written to JDE.
3
PO created through JDE Orchestration — P43 rules enforced
Write operations route exclusively through JDE Orchestrations, so every PO creation or update runs through P43 business logic exactly as your team has configured it. The AI agent cannot skip or bypass approval routing, cost thresholds, or any other business rule. The confirmed PO number is returned immediately.
4
Approval queue surfaces to approvers — in chat or dashboard
Approvers can ask "what POs are pending my approval?" — the agent queries the live approval queue from JDE and returns details. They can review, approve, or reject directly in conversation, or use the React web dashboard for a visual approval workflow. The full audit trail lives in JDE.
Why It's Different

Five capabilities that set it apart.

Not a chatbot wrapper. Not a screen-scraper. A purpose-built integration layer with real JDE business logic at every step.

Pillar 1
Conversational PO Management
"Create a PO for Acme Corp, 100 units of ABC123 at branch M30" works end to end through natural language. The agent handles validation, data lookup, and PO creation — returning a confirmed PO number. No JDE training. No form navigation.
Pillar 2
Approval Workflow Built-In
The approval queue is a first-class tool. Agents can list pending approvals, retrieve full PO details, approve, or reject — all from conversation. Approvers never need to log into JDE directly. The full audit trail is preserved in EnterpriseOne.
Pillar 3
Business Rules Enforced, Not Bypassed
Every write operation routes through JDE Orchestrations. P43 business logic runs exactly as configured — approval thresholds, routing rules, cost controls. AI agents can't accidentally shortcut your governance. Compliance is structural, not advisory.
Pillar 4
Two Interfaces, One Platform
AI agents interact through 18 MCP tools. Human users interact through the React web dashboard — a visual interface for PO tracking, approval queues, and status updates. Both backed by the same service, the same business logic, the same JDE data.
Pillar 5
Discoverable by Design
Layer 0 provides two dynamic discovery tools that let an AI agent introspect any JDE table the AIS server exposes — live, at runtime. The agent's capabilities aren't limited to pre-built tools: any exposed JDE table becomes explorable without additional development.
